# xv6 性能对比实验方案（修正版）

## 一、实验目标

**核心目的**：通过科学严谨的对比实验，量化内存管理和调度算法优化的性能提升。

**对比维度**：
- **baseline 分支**：原版 xv6（RR调度 + 链表分配器 + Eager Copy + Eager Allocation）
- **testing 分支**：优化版 xv6（多调度器 + 伙伴系统 + COW + 懒分配 + mmap）

---

## 三、COW (Copy-on-Write) 实验设计

### 3.1 核心对比

**Eager Copy vs COW**

| 实验场景 | baseline 行为 | testing 行为 |
|---------|--------------|-------------|
| fork 大内存进程 | 立即复制全部 | 延迟复制 |
| 子进程只读访问 | 复制全部（浪费） | 复制0页 |
| 子进程部分写入 | 复制全部 | 复制写入部分 |
| fork+exec | 复制后丢弃 | 零复制 |

### 3.2 测试用例

| 测试ID | 场景 | 操作 | 预期结果 | 特殊说明 |
|--------|------|------|---------|---------|
| COW-1 | fork+exec | 分配5MB，fork后立即exec，重复20次 | ↑90% | 最优场景 |
| COW-2 | fork只读 | 分配5MB，fork后只读取，重复20次 | ↑95% | 零复制 |
| COW-3 | fork部分写 | 分配5MB，fork后写30%，重复20次 | ↑60% | 中间场景 |
| COW-4 | fork全写 | 分配5MB，fork后全写入，重复20次 | ↓-15% | ⚠️ 反例：缺页开销超过收益 |

## 四、懒分配 (Lazy Allocation) 实验设计

### 4.1 核心对比

**Eager Allocation vs Lazy Allocation**

| 实验场景 | baseline 行为 | testing 行为 |
|---------|--------------|-------------|
| sbrk 扩展堆空间 | 立即分配全部页 | 仅更新大小 |
| 稀疏内存访问 | 已分配，无页错误 | 按需分配，触发页错误 |
| 不访问 | 已分配全部 | 分配0页 |

### 4.2 测试用例

| 测试ID | 场景 | 操作 | 预期结果 | 特殊说明 |
|--------|------|------|---------|---------|
| LAZY-1 | 稀疏访问 | sbrk 1MB，访问 10KB，重复100次 | ↑90% | 最优场景 |
| LAZY-2 | 半访问 | sbrk 1MB，访问 512KB，重复100次 | ↑40% | 中间场景 |
| LAZY-3 | 全访问 | sbrk 1MB，访问全部，重复100次 | ↓-15% | ⚠️ 反例：缺页风暴 |

**重要**：LAZY 测试必须在两个分支都跑，才能证明优化效果：
- baseline：sbrk 立即分配全部物理页 → 测量总时间
- testing：sbrk 只更新大小，访问时才分配 → 测量总时间
- 对比：testing 节省内存、速度快

## 五、伙伴系统 (Buddy System) 实验设计

### 5.1 核心对比

**链表分配器 vs 伙伴系统**

伙伴系统的核心价值是**内核层碎片控制**，不是用户态 malloc。

### 5.2 测试用例

| 测试ID | 场景 | 操作 | 测量指标 |
|--------|------|------|---------|
| BUDDY-1 | 内核碎片 | fork多进程制造内核碎片，测最大分配 | KB |

### 5.3 BUDDY-1 实现方案

**内核层碎片测试**（不是用户态 malloc 碎片）：

```c
void test_buddy_1_kernel_frag(void) {
    int pids[40];
    int sizes[] = {1, 2, 4, 8, 16};  // 页数
    
    // Phase 1: 创建多进程，分配不同大小内存
    for(int i = 0; i < 40; i++) {
        pids[i] = fork();
        if(pids[i] < 0) {
            printf("  [WARN] fork failed at %d\n", i);
            break;
        }
        if(pids[i] == 0) {
            sbrk(sizes[i % 5] * 4096);  // 触发内核分配
            sleep(50);
            exit(0);
        }
    }
    
    // Phase 2: 杀掉偶数进程，制造内核内存空洞
    for(int i = 0; i < 40; i += 2)
        kill(pids[i]);
    sleep(2);
    
    // Phase 3: 测量最大可分配块
    int max_kb = measure_max_alloc() / 1024;
    printf("RESULT:BUDDY_1_max_alloc:%d:0:kb\n", max_kb);
    
    // 清理
    for(int i = 1; i < 40; i += 2)
        kill(pids[i]);
    while(wait(0) > 0);
}
```

**预期结果**：
- baseline（链表）：碎片化严重，最大分配小
- testing（Buddy）：自动合并，最大分配大

## 七、调度器实验设计

### 7.1 核心机制：g_epoch

调度器测试的关键是**统一计时起点**：

```c
volatile int g_epoch = 0;

void spawn_sched_1_batch(void) {
    g_epoch = uptime();  // fork 前赋值
    for(int i = 0; i < 10; i++) {
        if(fork() == 0) {
            cpu_task_heavy();
            printf("PTURN:SCHED_1_%d:%d\n", i, uptime() - g_epoch);
            exit(0);
        }
    }
}
```

**原理**：
1. g_epoch 在 fork 前赋值
2. fork 复制地址空间，子进程能读到
3. fork 后不修改，不触发 COW
4. 所有子进程共享同一个起点，计时准确

### 7.2 测试用例

| 测试ID | 场景 | 任务组成 | 测量指标 | 对比调度器 |
|--------|------|---------|---------|-----------|
| SCHED-1 | 批处理吞吐 | 10个CPU密集任务 | 总完成时间 | RR vs FCFS |
| SCHED-2 | 护航效应 | 1长任务+9短任务 | 短任务周转时间 | RR vs FCFS |
| SCHED-3 | 优先级响应 | 3高优先+7低优先 | 高优先级周转时间 | RR vs PRIORITY |
| SCHED-4 | 饥饿现象 | 高优先级持续运行 | 低优先级运行次数 | RR vs PRIORITY |
| SCHED-5 | 公平性 | 10个相同任务 | 运行次数标准差 | RR vs LOTTERY |

### 9.3 公共辅助函数

```c
// ═══ 公共辅助函数（baseline 和 testing 都需要）═══

// 1. 二分法测最大分配
int measure_max_alloc(void) {
    int lo = 4096, hi = 16 * 1024 * 1024, best = 0;
    while(lo <= hi) {
        int mid = ((lo + hi) / 2) & ~(4096 - 1);
        char *p = sbrk(mid);
        if(p != (char*)-1) {
            best = mid;
            sbrk(-mid);
            lo = mid + 4096;
        } else {
            hi = mid - 4096;
        }
    }
    return best;
}

// 2. CPU 密集负载
void cpu_task_heavy(void) {
    volatile uint64 result = 0;
    for(int i = 0; i < 10000000; i++)
        result += (uint64)i * i;
}

void cpu_task_light(void) {
    volatile uint64 result = 0;
    for(int i = 0; i < 100000; i++)
        result += i;
}

// 3. 内存分配辅助（COW 测试用）
static char *mem_base = 0;
static int mem_size = 0;

void alloc_mem(int mb) {
    mem_size = mb * 1024 * 1024;
    mem_base = sbrk(mem_size);
    if(mem_base == (char*)-1) {
        printf("  [ERROR] sbrk failed\n");
        return;
    }
    // 写每一页确保分配
    for(int i = 0; i < mem_size; i += 4096)
        mem_base[i] = 'A';
}

void free_mem(void) {
    if(mem_base && mem_size > 0)
        sbrk(-mem_size);
}

// 4. 统计函数
#define WARMUP_RUNS 2
#define TEST_RUNS 10
#define MAX_RESULT 10

static int results[MAX_RESULT];
static int result_count = 0;

static void sort_results(void) {
    for(int i = 0; i < result_count - 1; i++) {
        for(int j = i + 1; j < result_count; j++) {
            if(results[j] < results[i]) {
                int tmp = results[i];
                results[i] = results[j];
                results[j] = tmp;
            }
        }
    }
}

static int calc_avg(void) {
    if(result_count < 3) return 0;
    int sum = 0;
    for(int i = 1; i < result_count - 1; i++)
        sum += results[i];
    return sum / (result_count - 2);
}

static int calc_std(int avg) {
    if(result_count < 3) return 0;
    int var = 0;
    for(int i = 1; i < result_count - 1; i++) {
        int diff = results[i] - avg;
        var += diff * diff;
    }
    int std = 0;
    int v = var / (result_count - 2);
    while(std * std < v) std++;
    return std;
}
```
