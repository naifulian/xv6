## 页面结构

前端的目标不是单页拼盘，而是“主页 + 子页面”的教学任务管理器结构。

核心页面包括：

- Home
- Processes
- Scheduler
- Memory
- Events

## 路由思想

每个页面只订阅自己需要的数据，不再把所有状态都塞进一个大刷新循环里。

这能把：

- 论文导览
- 进程详情
- 调度观察
- 内存视图
- 事件时间线

拆成更清楚的阅读层次。

## 刷新节奏

分层刷新比统一整页轮询更适合模块四，因为不同页面关注的数据粒度不一样。

这也是模块四从“慢速看板”收口到“任务管理器式界面”的关键原因。
