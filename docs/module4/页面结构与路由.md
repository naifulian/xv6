## 页面结构

前端目标已经从“单页拼盘”收口为“左侧导航 + 右侧工作区”的子页面结构。

当前页面包括：

- Processes
- Scheduler
- Memory
- Events
- Home（概览页，不再作为默认落地页）

## 路由思想

页面切换通过左侧导航直接进入子页面，不再要求用户先下滑读首页再跳转。

默认路由调整为：

- `#/processes`

概览页仍然保留，但定位改为总览面板，而不是强制入口页。

## 刷新节奏

每个页面继续只订阅自己需要的数据：

- `Processes` 轮询进程表和选中进程详情。
- `Scheduler` 轮询调度摘要和热点进程。
- `Memory` 轮询内存摘要并复用进程详情。
- `Events` 单独承接高频事件刷新。

这样可以保持任务管理器式界面，不再退回整页拼盘和整页叙事。

## 2026-03-26 界面重构补充

- 顶部大段说明文案已移除，页面信息改为数据卡片、表格和时间线。
- 左侧导航按钮固定承载页面切换，右侧只显示当前页内容。
- `Home` 页改名为“概览”，只保留简短摘要，不再承担答辩导览文案。
