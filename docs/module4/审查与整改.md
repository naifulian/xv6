# 模块四审查与整改

## 当前结论

模块四目前还不应按“完成态”来写，更准确的描述是：

- 数据入口方向已经对；
- 监控导出链路已有雏形；
- 真实剩余工作主要在展示层工程化，而不是底层数据采集思路。

## 已完成项

1. 明确模块四应优先消费模块三产物，而不是重新解析原始日志。
2. 补上了 `sysmon` 这类面向监控的结构化导出工具。
3. 修复了用户态监控工具与新增调度器不同步、以及 `sysmon` 栈溢出等基础问题。
4. 默认 QEMU 启动链路已补齐 telemetry side channel 设备挂载，`telemetrywrite()` 不再依赖手工补 `QEMU_EXTRA_OPTS` 才能走通。

## 仍待完成项

1. 页面结构、图表编排、状态提示和交互体验尚未统一。
2. 监控快照到前端页面的完整工作流还需要进一步收口。
3. 模块四文档当前以方案说明为主，不应过度承诺“已实现完整 Web 平台”。

## 最终判断

模块四现在最适合扮演“展示层接收器”：稳定复用模块三结果，适度接入系统快照，而不是在项目尾声再引入复杂的新数据格式。
