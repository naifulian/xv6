## 写作规则

1. 总架构文档负责建立整体认识，不承载完整排障细节。
2. 模块文档负责讲透原理设计、接口与验证，不再停留在摘要层。
3. `docs/bug/` 保留完整 bug 详案，主文档只吸收精简闭环。
4. 正式实验口径统一以 `experiments/logs/thesis_final` 为准。
5. 对高波动项应说明 CI、CV 和 warning 所表达的是解释边界，而不是简单忽略异常。
