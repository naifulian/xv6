# BUG-011: 引用计数未初始化导致 fork 卡死

## 基本信息

| 项目 | 内容 |
|------|------|
| 编号 | BUG-011 |
| 分类 | 内存模块 (MEM) |
| 严重程度 | 高 |
| 发现时间 | 2026-02-15 |
| 修复提交 | `1d240ac` |
| 影响文件 | `kernel/kalloc.c` |

## 问题描述

`kalloc()` 分配新页面时没有初始化 `ref_count[]`，导致引用计数为 0，`ref_dec()` 错误地认为应该释放页面，造成 fork 卡死。

## 问题代码

```c
void* kalloc(void)
{
    // ... 分配页面
    // 缺少引用计数初始化
    return (void*)r;
}
```

## 问题分析

1. **现象**：forktest 卡死
2. **调试过程**：
   - 在 `ref_dec()` 添加打印
   - 发现引用计数为 0，页面被错误释放
3. **根本原因**：`kalloc()` 分配新页面时没有初始化 `ref_count[]`

## 影响范围

- fork 系统调用卡死
- COW 功能异常
- 系统不稳定

## 修复方案

在 `kalloc()` 中初始化引用计数为 1。

## 修复后代码

```c
void* kalloc(void)
{
    // ... 分配页面
    
    if(r) {
        int idx = pa2index((uint64)r);
        if(idx >= 0) {
            ref_count[idx] = 1;  // 初始化引用计数为 1
        }
    }
    
    return (void*)r;
}
```

## 反省

1. **初始化遗漏**：新分配的资源需要初始化
2. **测试覆盖不足**：没有充分测试 fork 场景
3. **改进建议**：分配函数必须初始化相关元数据

## 影响评估

- 功能影响：重大，修复后 fork 正常
- 稳定性：显著提高
- COW 功能：正常工作
