# BUG-005: 测试数量硬编码

## 基本信息

| 项目 | 内容 |
|------|------|
| 编号 | BUG-005 |
| 分类 | 测试模块 (TEST) |
| 严重程度 | 低 |
| 发现时间 | 2026-03-05 |
| 修复提交 | `dee166f` |
| 影响文件 | `tests/src/test_runner.c` |

## 问题描述

测试数量 `NUM_TESTS` 被硬编码为 60，添加新测试时需要手动更新，容易遗漏。

## 问题代码

```c
#define NUM_TESTS 60  // 硬编码

struct test_case tests[] = {
    // ... 60 个测试用例
};

int num_tests = NUM_TESTS;
```

## 问题分析

1. **维护困难**：添加新测试需要手动更新
2. **容易出错**：可能忘记更新导致新测试不被执行
3. **代码耦合**：测试数量与测试数组分离

## 影响范围

- 添加新测试时可能遗漏更新
- 测试框架可维护性差

## 修复方案

使用 `sizeof` 自动计算测试数量。

## 修复后代码

```c
struct test_case tests[] = {
    // ... 测试用例
};

#define NUM_TESTS (sizeof(tests) / sizeof(tests[0]))
int num_tests = NUM_TESTS;
```

## 反省

1. **设计不够灵活**：没有考虑自动计算
2. **维护性考虑不足**：没有预见到后续扩展
3. **改进建议**：数组大小应该自动计算，避免硬编码

## 影响评估

- 功能影响：无
- 可维护性：显著改善
- 扩展性：提高
