# BUG-004: Lottery 调度器随机种子固定

## 基本信息

| 项目 | 内容 |
|------|------|
| 编号 | BUG-004 |
| 分类 | 调度模块 (SCHED) |
| 严重程度 | 低 |
| 发现时间 | 2026-03-05 |
| 修复提交 | `8fa0c4e` |
| 影响文件 | `kernel/sched/sched_lottery.c` |

## 问题描述

随机种子使用固定值初始化，每次系统启动时调度序列可能相同，降低了彩票调度的随机性。

## 问题代码

```c
static uint64 seed = 12345;  // 固定种子
```

## 问题分析

1. **可预测性**：每次系统启动，调度序列相同
2. **公平性问题**：在特定场景下可能导致某些进程总是被选中
3. **违背设计初衷**：彩票调度应该具有随机性

## 影响范围

- 单核环境下调度序列可预测
- 可能影响长时间运行进程的公平性
- 影响彩票调度的测试验证

## 修复方案

使用系统 ticks 作为种子源，首次调用时动态初始化。

## 修复后代码

```c
static struct proc*
lottery_pick_next(void)
{
    struct proc *p;
    struct cpu *c = mycpu();
    int total_tickets = 0;
    static uint64 seed = 0;
    static int initialized = 0;

    if(!initialized) {
        seed = ticks * 0x9e3779b97f4a7c15ULL;  // 使用 ticks 初始化
        initialized = 1;
    }

    // ... 其余代码
}
```

## 反省

1. **随机性理解不足**：没有意识到固定种子的影响
2. **测试场景单一**：没有进行多次启动的对比测试
3. **改进建议**：涉及随机性的功能需要考虑种子初始化

## 影响评估

- 功能影响：无
- 公平性：改善
- 随机性：显著提高
