# BUG-007: 调度器切换缺少锁保护

## 基本信息

| 项目 | 内容 |
|------|------|
| 编号 | BUG-007 |
| 分类 | 调度模块 (SCHED) |
| 严重程度 | 高 |
| 发现时间 | 2026-02-15 |
| 修复提交 | `1d240ac`, `d931fce` |
| 影响文件 | `kernel/sched/sched.c` |

## 问题描述

调度器切换时没有锁保护，可能导致竞态条件。

## 问题代码

```c
void setscheduler(int policy)
{
    // 缺少锁保护
    current_sched_ops = sched_ops_table[policy];
}
```

## 问题分析

1. **竞态条件**：多核环境下同时切换调度器可能导致不一致
2. **数据竞争**：`current_sched_ops` 被多个 CPU 同时访问
3. **可能导致崩溃**：调度器切换过程中被中断

## 影响范围

- 多核环境下可能出现调度错误
- 可能导致系统不稳定
- 数据竞争

## 修复方案

添加自旋锁保护调度器切换。

## 修复后代码

```c
static struct spinlock sched_lock;

void setscheduler(int policy)
{
    acquire(&sched_lock);
    current_sched_ops = sched_ops_table[policy];
    release(&sched_lock);
}
```

## 反省

1. **并发安全意识不足**：没有考虑多核场景
2. **锁设计不完整**：关键操作缺少锁保护
3. **改进建议**：所有全局共享数据的修改都需要锁保护

## 影响评估

- 功能影响：重大，修复后避免竞态
- 稳定性：显著提高
- 多核安全：确保
