# BUG-009: free(NULL) 安全检查缺失

## 基本信息

| 项目 | 内容 |
|------|------|
| 编号 | BUG-009 |
| 分类 | 内存模块 (MEM) |
| 严重程度 | 中 |
| 发现时间 | 2026-02-15 |
| 修复提交 | `1d240ac` |
| 影响文件 | `user/umalloc.c`, `tests/src/test_buddy.c` |

## 问题描述

`free()` 函数没有检查 NULL 指针，可能导致崩溃。

## 问题代码

```c
void free(void *ap)
{
    // 缺少 NULL 检查
    Header *bp, *p;
    bp = (Header*)ap - 1;  // NULL - 1 导致未定义行为
    // ...
}
```

## 问题分析

1. **未定义行为**：NULL 指针减 1 导致未定义行为
2. **不符合标准**：C 标准要求 free(NULL) 是安全操作
3. **可能导致崩溃**：访问非法内存

## 影响范围

- 调用 free(NULL) 会崩溃
- 不符合 C 标准行为
- 影响代码健壮性

## 修复方案

添加 NULL 检查。

## 修复后代码

```c
void free(void *ap)
{
    if(ap == 0)  // 添加 NULL 检查
        return;

    Header *bp, *p;
    bp = (Header*)ap - 1;
    // ...
}
```

## 反省

1. **标准遵循不足**：没有遵循 C 标准对 free(NULL) 的要求
2. **健壮性考虑不足**：没有处理边界情况
3. **改进建议**：所有公共 API 都应该处理 NULL 输入

## 影响评估

- 功能影响：重大，修复后避免崩溃
- 健壮性：显著提高
- 标准兼容性：符合 C 标准
