# 模块二总览与边界

模块二负责把模块一的关键机制转成可回归验证的测试框架。

这份文档只看最上层的定位和边界：

1. 为什么测试要单独成为一个模块。
2. 为什么模块二要同时保留强断言和 smoke 场景。

详细组织方式仍然保留在：

- [模块二架构设计](/home/niya/xv6/docs/module2/架构设计.md)

## 模块二的任务

模块二不是单纯“多写几个测试”，而是建立一个稳定的验证平面：

- 为模块一提供统一测试入口。
- 为模块三提供正确性护栏。
- 避免测试框架自己误解 xv6 语义。

## 两条边界

1. 强断言测试负责证明关键路径真的成立。
2. smoke 测试负责说明系统至少能稳定运行并保留基本语义。

这也是为什么模块二不会把所有场景都写成“完全证明”，而是会诚实地区分可强断言和只能 smoke 的部分。
