# 模块三总览与边界

模块三负责把模块一和模块二的结果转成论文可用的实验证据。

这份文档只抓最上层的两个边界：

1. 为什么模块三是实验方法模块，而不是脚本堆。
2. 为什么正式内存实验只保留 `COW + Lazy`。

完整的方法学说明仍然保留在：

- [模块三架构设计](/home/niya/xv6/docs/module3/架构设计.md)

## 模块三的职责

模块三要解决的不是“怎么自动跑实验”，而是：

- 实验问题有没有问准。
- 数据链路能不能审计。
- 最终结论能不能写进论文。

## 两个边界

1. `COW + Lazy` 是当前正式内存实验主线。
2. Buddy 和 `mmap` 保留实现，但不再作为 formal comparison 主证据。

这不是放弃，而是为了让论文结论更稳。
