# 数据链路与门禁

模块三这部分只看“数据怎么从测试变成论文产物”。

完整方法仍然保留在：

- [模块三架构设计](/home/niya/xv6/docs/module3/架构设计.md)

## 数据链路

```text
user/perftest.c
-> experiments/collect/collect_data.sh
-> experiments/analyze/experiment_data.py
-> experiments/analyze/validate_logs.py
-> experiments/analyze/plot_results.py
```

这条链路的重点不是“自动化”，而是每一步都能审计。

## 门禁职责

`validate_logs.py` 的角色是正式结论前的质量门禁，重点检查：

- 缺失测试项。
- 混合 signature / command / commit。
- partial run 和 raw-only run。
- 低分辨率、低轮数、高波动。

## 产物口径

模块三最终要交付的是：

- 结构化数据集。
- 可复采日志。
- 可直接写入论文的图表与结论。

所以这部分更像“方法学中台”，而不是单纯的脚本集合。
