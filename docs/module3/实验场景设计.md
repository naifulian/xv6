# 实验场景设计

模块三这部分只看“场景怎么设计”，不看完整数据管线。

它主要回答三个问题：

1. COW 在什么写入比例下更有收益。
2. Lazy 在什么访问密度下更有收益。
3. 调度策略在什么 workload 下能看见可解释差异。

完整方法仍然保留在：

- [模块三架构设计](/home/niya/xv6/docs/module3/架构设计.md)

## COW 场景

- `COW_NO_ACCESS`
- `COW_READONLY`
- `COW_PARTIAL`
- `COW_FULLWRITE`

这组场景的目的不是证明 COW 总是更快，而是证明收益依赖写入比例。

## Lazy 场景

- `LAZY_SPARSE`
- `LAZY_HALF`
- `LAZY_FULL`

这组场景的目的不是证明 Lazy 永远占优，而是说明收益依赖访问密度。

## 调度场景

- `throughput`
- `convoy`
- `fairness`
- `response`

它们各自对应混合批处理、convoy effect、公平性和新到短任务响应延迟。
