# 模块一总览与边界

模块一是整个项目的基础层，负责把原始 xv6 扩展为一个可以支撑调度比较、内存机制分析、回归测试和后续展示的增强型内核。

这份文档只抓最上层的两个问题：

1. 为什么模块一值得单独作为一个毕设模块。
2. 模块一的调度主线和内存主线分别边界在哪里。

如果你想先建立整体感觉，再进入细节，建议按下面顺序读：

1. [调度框架设计](/home/niya/xv6/docs/module1/调度框架.md)
2. [内存语义设计](/home/niya/xv6/docs/module1/内存语义.md)
3. [模块一架构设计](/home/niya/xv6/docs/module1/架构设计.md)

## 模块一为什么重要

原始 xv6 很适合教学，但它只有单一 RR 调度和相对朴素的地址空间管理。只靠原始版本，很难同时支撑：

- 多策略调度比较。
- COW、Lazy、匿名私有 `mmap` 的统一语义。
- 回归测试和实验采样。
- 后续展示层的数据消费。

所以模块一不是“多加几个功能”那么简单，而是在控制平面和地址空间语义上完成一次系统级升级。

## 本模块的两个边界

1. 调度主线负责统一调度框架、策略切换和生命周期钩子。
2. 内存主线负责统一地址空间语义、缺页路径和页面权限落地。

Buddy、COW、Lazy 和 `mmap` 仍然都在仓库里，但论文主线更强调它们如何被统一到同一套语义中，而不是把它们拆成彼此割裂的点状特性。
