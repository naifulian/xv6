# 内存语义设计

模块一的内存主线要回答的是“如何把 COW、Lazy Allocation 和匿名私有 `mmap` 统一到同一套地址空间语义里”。

这份文档只抓地址空间与缺页语义的主干，不展开全部实现细节。完整说明仍然保留在：

- [模块一架构设计](/home/niya/xv6/docs/module1/架构设计.md)
- [内存管理优化架构设计](/home/niya/xv6/docs/module1/内存管理优化.md)

## 核心问题

1. `heap_end + vmas[] + sz` 为什么比单一 `sz` 更适合教学化增强内核。
2. 为什么 COW、Lazy 和 `mmap` 必须共用统一缺页入口。
3. 为什么 `munmap`、权限落地和回收路径必须一起设计。

## 语义主线

模块一内存部分真正重要的不是某个 feature 单点，而是这三层语义如何互相配合：

- 进程地址空间元数据：`heap_end`、`vmas[]`、`sz`
- 缺页处理主入口：`vmfault()`
- 权限与回收落地：`copyout()`、`munmap()`、`mmap()`

这三层合起来，才让内存增强从“补丁集合”变成“统一地址空间模型”。
