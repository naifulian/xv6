## 已完成里程碑

### 模块一

- 实现了 `RR/FCFS/PRIORITY/SML/LOTTERY/SJF/SRTF/MLFQ/CFS` 九种调度策略。
- 实现了 Buddy System、Copy-on-Write、Lazy Allocation、匿名私有 `mmap` 原型。
- 调度框架与虚拟内存路径已经过一轮审查和整改，不再停留在“功能点拼接”状态。

### 模块二

- 建立了 `test_runner` 统一入口。
- 将 68 个用例组织为 Buddy、mmap、COW、Lazy、Integration、Boundary、Scheduler 七类。
- 修正了 xv6 `wait()` 语义、lazy 接口误测、调度顺序断言偏弱等问题。

### 模块三

- 建立了 `collect -> validate -> plot` 的实验链路。
- 正式数据集统一为 `experiments/logs/thesis_final`。
- baseline/testing 各 3 轮 complete runs 已完成。
- `validate_logs.py` 已清除 missing tests、mixed-signature、low-resolution、cross-branch signature 等结构性问题。
- 当前剩余 warning 主要是高波动项提醒，属于 QEMU+xv6 单核 tick 级噪声，不是链路错误。

### 模块四

- 模块三产物已迁移到 `experiments/outputs/`，后续展示层不再以 `webui/` 作为正式模块三目录。
- 已实现 `sysmon` 到结构化监控数据的基础导出链路。
- 目前仍处于“方案清晰、展示层待系统化完成”的阶段。
